# TurtleBot3 거리 인식/계산 참고 메모

본 문서는 즉시 코드 반영용이 아니라, 이후 설계/튜닝 시 참고하기 위한 정리 문서입니다.

## 1) LiDAR(LDS) 센서를 통한 직접 거리 측정

- TurtleBot3 LDS(예: LDS-01/02)는 대략 `0.12m ~ 3.5m` 범위의 물체를 스캔합니다.
- 다른 TurtleBot은 LiDAR 관점에서 장애물로 관측됩니다.
- 근접 안전 동작(정지/감속/회피) 임계값(예: 0.5m)을 설정해 운용할 수 있습니다.

## 2) ROS 위치 데이터(/odom) 기반 거리 계산

- 각 로봇이 발행하는 위치를 같은 좌표계로 정렬한 뒤, 유클리드 거리로 계산합니다.
- 기본 공식:
  `distance = sqrt((x1 - x2)^2 + (y1 - y2)^2)`
- Gazebo 등 시뮬레이션에서도 동일한 방식으로 거리 추정이 가능합니다.

## 3) 주요 제약 사항

- LiDAR 최소 인식 거리 근처(약 0.12m 부근)는 오차/미검출 가능성이 있습니다.
- 센서 가림(로봇 구조물, 주변 물체, 각도)에 의해 측정이 불안정할 수 있습니다.
- `/odom` 기반 계산은 두 로봇 좌표계가 연결되지 않으면 잘못된 거리값이 나올 수 있습니다.

## 4) 권장 혼용 전략

- 중/원거리 추종: TF/odom 기반 거리 계산 사용
- 근접 안전 제어: LiDAR 최소거리 기반 감속/정지 보강
- 결론: 위치 기반 추종 + LiDAR 안전 임계값을 함께 쓰는 하이브리드 방식 권장

## 5) 현재 프로젝트에 적용 시 체크포인트

- `TB3_1`/`TB3_2` 프레임 연결(TF tree) 유효성 확인
- 추종 프레임(`tracking_frame`)과 costmap `global_frame` 일치 확인
- LiDAR 기반 임계값 사용 시 노이즈 필터(평균/중앙값/점프 제한) 적용 검토

## 6) 현재 코드 기준 적용 상태 (2026-03)

- follower 목표점 생성:
  - 리더 위치 + 리더 yaw를 사용해, 리더 뒤 `follow_distance` 오프셋 지점을 목표점으로 생성
- 추종 실행:
  - 목표점을 포함한 2-point 경로를 Nav2 `FollowPath`로 전송
- 업데이트 제어:
  - `goal_update_distance_threshold`, `goal_update_min_period_sec`로 goal 과전송 제한
